# API reference

The Hosaka backend speaks one versioned JSON API at `/api/v1/*`. Every
response is a Pydantic model, so the OpenAPI schema below is the contract.

!!! tip
    The schema is regenerated from the source on every commit by
    `scripts/dump_openapi.py` and rendered live below by
    [mkdocs-render-swagger-plugin].

## Auth

| Source | Behaviour |
|---|---|
| `127.0.0.1` (loopback — kiosk SPA, TTY tools) | always allowed |
| `Authorization: Bearer $(cat /etc/hosaka/api-token)` from LAN | allowed |
| LAN without token | reads OK, writes return `403` |

The token is generated by the installer with `openssl rand -hex 32`. Rotate
it any time by overwriting `/etc/hosaka/api-token` and restarting
`hosaka-webserver.service`.
