# Claims audit

The full audit lives in
[`docs/CLAIMS_AUDIT.md`](https://github.com/example/Hosaka/blob/main/docs/CLAIMS_AUDIT.md).
That document was generated by an automated pass over the repo and
categorises every claim we make in READMEs / install scripts as one of:

- **WORKS** — actually does what it says
- **PARTIAL** — works but with caveats
- **BROKEN** — claim doesn't match reality, file a bug
- **ASPIRATIONAL** — TODO masquerading as docs
- **MISLEADING** — slightly true but the marketing oversells

Use it as the canonical "what does this codebase actually do" checklist.

## Open work — top picks

These are the items the audit flagged that still need follow-through:

1. AP-fallback wifi mode (`hostapd` + captive portal) for fresh installs with
   no network.
2. `/messages` webhooks landing page in the SPA still references the old
   external service.
3. Picoclaw model selection round-trip in the kiosk Settings drawer.
4. Plant feature persistence across reboots.
5. Tailscale auto-join — currently you have to `tailscale up` by hand.

If you're picking up work, that's a good ordered list to start from.
